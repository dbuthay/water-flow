**Class: `IrrigationMonitorOptionsFlowHandler(config_entries.OptionsFlow)`**

Instance variables (set during flow):
- `_new_zone_ids: list[str]` — valve entity IDs selected in init step
- `_zone_iterator: list[str]` — copy of _new_zone_ids for iterating per-zone steps
- `_zone_settings: dict[str, dict]` — accumulated per-zone settings
- `_new_flume_entity_id: str` — updated Flume entity ID
- `_new_poll_interval: int` — updated poll interval

**Method: `async_step_init(self, user_input: dict | None = None)`**
- If user_input is not None:
  - Store `self._new_flume_entity_id = user_input[CONF_FLUME_ENTITY_ID]`
  - Store `self._new_poll_interval = user_input[CONF_POLL_INTERVAL]`
  - Store `self._new_zone_ids = user_input[CONF_MONITORED_ZONES]`
  - Initialize `self._zone_settings = {}`
  - Initialize `self._zone_iterator = list(self._new_zone_ids)`
  - Return `await self.async_step_zones()`
- Otherwise show form with:
  - `CONF_FLUME_ENTITY_ID`: EntitySelector with integration="flume" filter, default from `self.config_entry.data.get(CONF_FLUME_ENTITY_ID)`
  - `CONF_MONITORED_ZONES`: SelectSelector with valve candidates from `_discover_valve_entities()`, multiple=True, default from `self.config_entry.data.get(CONF_MONITORED_ZONES, [])`
  - `CONF_POLL_INTERVAL`: NumberSelector(NumberSelectorConfig(min=10, max=300, step=5, unit_of_measurement="s")), default from `self.config_entry.data.get(CONF_POLL_INTERVAL, DEFAULT_POLL_INTERVAL)`

**Method: `_discover_valve_entities(self) -> list[tuple[str, str]]`**
- Same logic as ConfigFlow._discover_valve_entities (can be extracted to a module-level function to share)

**Method: `async_step_zones(self, user_input: dict | None = None)`**
- If user_input is not None AND there is a current zone being configured:
  - Store user_input settings for current zone in `self._zone_settings`
  - Pop current zone from `self._zone_iterator`
- If `self._zone_iterator` is empty (all zones configured or no zones):
  - Apply the CRITICAL merge pattern:
    ```python
    existing = dict(self.config_entry.options)
    old_zones = existing.get(CONF_ZONES, {})
    updated_zones = {}
    for zone_id in self._new_zone_ids:
        if zone_id in self._zone_settings:
            # User provided new settings for this zone
            base = old_zones.get(zone_id, {})
            base.update(self._zone_settings[zone_id])
            updated_zones[zone_id] = base
        elif zone_id in old_zones:
            # Existing zone not re-configured — preserve entirely
            updated_zones[zone_id] = old_zones[zone_id]
        else:
            # Brand new zone — apply defaults
            updated_zones[zone_id] = {
                CONF_SHUTOFF_ENABLED: DEFAULT_SHUTOFF_ENABLED,
                CONF_ALERTS_ENABLED: DEFAULT_ALERTS_ENABLED,
                CONF_CALIBRATED_FLOW: None,
                CONF_THRESHOLD_MULTIPLIER: DEFAULT_THRESHOLD_MULTIPLIER,
            }
    existing[CONF_ZONES] = updated_zones
    return self.async_create_entry(data=existing)
    ```
  - NOTE: Removed zones (those in old_zones but NOT in self._new_zone_ids) are simply not carried into updated_zones — this clears their data entirely per CONTEXT.md decision.
- If zones remain:
  - Get current zone = `self._zone_iterator[0]`
  - Get existing settings for this zone from `self.config_entry.options.get(CONF_ZONES, {}).get(zone_id, {})`
  - Show form with step_id="zones":
    - `CONF_SHUTOFF_ENABLED`: BooleanSelector(), default from existing or DEFAULT_SHUTOFF_ENABLED

    - `CONF_THRESHOLD_MULTIPLIER`: NumberSelector(NumberSelectorConfig(min=1.0, max=5.0, step=0.1)), default from existing or DEFAULT_THRESHOLD_MULTIPLIER
  - Description placeholders should show the zone name

Also update `self.config_entry.data` with the new flume_entity_id, monitored_zone_entity_ids, and poll_interval by calling:
